## Reel Generation Pipeline (rebuilt for premium/viral quality)

The generation pipeline (`lib/generation-pipeline.ts`) now produces REAL composited vertical MP4s (1080x1920, h264+aac) via the Abacus FFmpeg API, replacing the old mock/stock-loop flow. Flow: script -> images -> voice -> music -> composite.

### Provider architecture (`lib/providers/`)
- `index.ts` exports `getScriptProvider` (LLMScriptProvider, model env SCRIPT_MODEL default gpt-5.4), `getImageProvider` (AbacusImageProvider, model env IMAGE_MODEL default gemini-3.1-pro, 9:16), `getVoiceProvider` (ElevenLabsVoiceProvider), `getMusicProvider` (LibraryMusicProvider).
- `fallback.ts`: `buildTemplateScript()` (curated affirmation templates w/ per-scene cinematic image prompts + timings) and `fallbackSceneImages()` (rehosts bundled /public/styles/*.jpg to public S3, rotated across scenes).
- Every step is wrapped in try/catch with graceful fallback so reels are ALWAYS produced.

## Stripe Subscription Activation (CRITICAL fix — June 2026)
- **Root cause of "stuck on Free after paying":** As of Stripe API `2025-03-31.basil`, `current_period_start`/`current_period_end` were REMOVED from the top-level Subscription object and moved to `subscription.items.data[0].current_period_*`. The account default API version is newer than the SDK pin (`2024-12-18.acacia` in `lib/stripe.ts`), so webhook events delivered `undefined` top-level fields → `new Date(undefined*1000)` = Invalid Date → Prisma upsert threw → subscription never updated → user stayed Free.
- **Fix:** `lib/stripe-sync.ts` exports `applySubscriptionUpdate(userId, subscription)` (shared, idempotent). Reads billing period from `items.data[0].current_period_*` first, falls back to top-level then start_date, derives end from interval if missing. Guards `typeof === 'number' && > 0`. Used by BOTH the webhook (`customer.subscription.created/updated` AND `checkout.session.completed`) and the new fallback endpoint.
- **Fallback endpoint:** `app/api/payments/verify-session/route.ts` (POST {sessionId}) — called by dashboard after Checkout redirect (`?session_id={CHECKOUT_SESSION_ID}` added to success_url in create-checkout + buy-credits). Ownership-checked + idempotent. Activates subs OR completes coin purchases even if webhook lags.
- Do NOT bump the Stripe SDK apiVersion — the fix handles both old & new shapes.
- Backfilled 2 real test users who paid before the fix (pro cmq0djw1q0007pd081pprhoal, premium cmq0d0mbz0000pd08tgbxrd1h) via applySubscriptionUpdate against their live Stripe subs — confirmed valid dates + correct tier in DB.
- Pricing display fix: landing-page annual /mo uses `Math.floor(t.monthly*50)/100` → Pro $9.99, Premium $24.99 (matches Settings). Phone-field validation on Stripe Link is a Stripe-hosted-checkout UX issue, NOT app-fixable.
