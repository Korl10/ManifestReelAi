## Reel duration pipeline (CRITICAL root-cause knowledge)
- `scriptPlanForDuration` in `lib/providers/llm-script.ts` previously had a hard floor of `Math.max(10, ...)` ŌåÆ 16 words / 4 scenes min ŌåÆ ~8s narration even for short reels. THAT was the bug that forced the old 5s reels to render ~8s and made the duration check unreliable.
- FIX: short-reel branch ŌĆö `if (target <= 9) { wordBudget = max(9, round((target-2.0)/0.55)); return { affirmationLines: 2, totalScenes: 3, wordBudget }; }`. Narration lands UNDER target; pipeline then EXTENDS the final scene to exactly the target ŌåÆ delta=0.
- `lib/generation-pipeline.ts` duration check (~line 625): ┬▒1s gate is ENABLED for ALL tiers now (removed the old free-tier skip). Verified delta=0 for 7s free reels.

## Thumbnails (real first-frame, not generic template)
- After final MP4 render, `extractPosterFrame(videoUrl)` in `lib/compositor.ts` extracts a representative frame via the FFmpeg API and stores the CDN URL in `Reel.thumbnailUrl`.
- WORKING FFmpeg command: `-i {{in_1}} -vf "thumbnail=100,scale=540:-2" -frames:v 1 -q:v 3 {{out_1}}`. IMPORTANT: `-ss` seeking (input OR output) FAILS on the Abacus FFmpeg API ("Output file out_1 not found") ŌĆö use the `thumbnail` filter instead, double-quoted in the command string.
- API precedence: `app/api/reels/[id]/route.ts` and `app/api/reels/route.ts` prefer the real `thumbnailUrl` over the generic style poster via `isPlaceholderUrl()`.
- Dashboard "My Reels" cards + library cards render `reel.thumbnailUrl` as an `aspect-[9/16]` `<img>` with a Film-icon fallback.
- Composited videoUrl is a permanent `cdn.abacus.ai/videos/...mp4`; extracted thumbnails are `cdn.abacus.ai/images/...jpg`.

## Music library infra
- **Vol. 1**: `lib/music-library.ts` + `public/music-library/manifest.json`: 40 tracks (28-190s, avg 38s) + 5 stingers. Weighted random matcher by mood/energy/category. Local files in `/public`, uploaded to S3 at generation time via `ensurePublicLocalAsset`.
- **Vol. 2**: `LibraryTrack` Prisma model. Uploaded via admin Music Loader ŌåÆ S3 cloud storage. FFmpeg processing normalizes EVERY track to exactly 60s: `-stream_loop -1 -i input -t 60` (loop-fills short tracks, trims long), + 0.5s fade-in + 0.8s fade-out at 59.2s + LUFS -14 + 192k MP3. All masters are uniform 60s. DB stores `cloudUrl` (processed) + `originalUrl` (raw). No local files needed. Minimum accepted Suno output: Ōēź35s.
- **Smart matcher** (`matchTrackAsync`): combines Vol. 1 manifest tracks + Vol. 2 DB tracks. Scoring: mood (via MOOD_ALIASES) ŌåÆ style overlap ŌåÆ platform fit ŌåÆ energy fit ŌåÆ provenance. `isCloudTrack()` helper tells pipeline whether to use `ensurePublicLocalAsset` (local) or direct URL (cloud).
- **Category mapping** (`lib/music-categories.ts`): 8 categories (spiritual, luxury, motivational, mysterious, historical, biblical, educated, meditation) ŌåÆ mood/style/energy defaults. `parseTrackFilename()` parses `{category}_{nn}_{descriptor}.mp3`.
- **Admin Music Loader** (`/admin/music-loader`): drag-drop MP3s, filename auto-parse, dry-run mode (preview tags without commit), batch upload+process. API at `/api/admin/music-loader` (GET: list, POST: upload+process) + `/api/admin/music-loader/presign` (presigned URLs).
- **Pipeline integration**: `generation-pipeline.ts` uses `matchTrackAsync` (async, includes DB tracks). V2 cloud tracks skip `ensurePublicLocalAsset` ŌĆö their `cloudUrl` is already a public S3 URL usable by the FFmpeg compositor.
- Compositor uses `-stream_loop 4` for music input ŌåÆ loops track if shorter than reel. `amix=inputs=2:duration=longest` + fade-out at T-0.8s handles truncation naturally.
